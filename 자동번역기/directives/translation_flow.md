# 번역 로직 지침서 (SOP)

이 지침서는 자동번역기 프로젝트의 핵심 번역 흐름 및 토큰 절감 전략을 설명합니다.

## 1. 번역 프로세스 흐름
사용자가 한국어 문장을 입력하면 다음 순서로 처리가 진행됩니다.

### Step 1: 시맨틱 캐싱 (Semantic Caching)
- 입력된 문장과 지식 베이스(`kb_data.json`) 내의 문장을 비교합니다.
- `difflib.SequenceMatcher`를 사용하여 유사도를 계산합니다.
- **유사도 >= 95%**: 기존 번역 데이터를 즉시 반환 (AI 호출 없음, 토큰 절감 100%).

### Step 2: RAG 기반 프롬프트 구성
- 캐시 히트가 되지 않았으나 유사도가 일정 수준(예: 50% 이상)인 경우, 가장 유사한 문장 쌍을 프롬프트에 '참고 스타일'로 주입합니다.
- 이를 통해 기존 번역 톤앤매너를 유지합니다.

### Step 3: 멀티 타겟 번역 (LLM)
- Gemini API (`gemini-flash-latest`)를 호출하여 영/일/중 번역을 동시에 수행합니다.
- JSON 모드를 사용하여 구조화된 데이터를 획득합니다.

## 2. 기술 설정
- **모델**: `gemini-flash-latest` (쿼터 및 가용성 최적화)
- **폴백**: 오류 발생 시 `gemini-pro-latest` 등으로 자동 전환
- **포맷**: 영, 일, 중 번역본을 포함한 JSON 객체
