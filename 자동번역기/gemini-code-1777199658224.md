# 📋 자동번역기 프로젝트 기획 요약

기존 번역 자산을 활용하여 토큰 사용량을 최소화하는 실시간 다국어(영·일·중) 번역 서비스입니다.

## 1. 핵심 목표
- **비용 최적화**: 시맨틱 캐싱 및 RAG 기술을 통한 AI 토큰 소모 최소화
- **정밀도 향상**: 기존 번역 데이터(Knowledge Base)를 학습/참조하여 일관된 스타일 유지
- **실시간성**: 안티그래비티 환경에 최적화된 저지연(Low-latency) 번역 제공

## 2. 토큰 절감 전략 (Token Saving Strategy)
| 단계 | 기술 명칭 | 설명 | 토큰 절감 효과 |
| :--- | :--- | :--- | :--- |
| **Step 1** | **시맨틱 캐싱** | 입력 문장과 기존 DB 내 유사도가 95% 이상일 경우 AI 호출 없이 즉시 반환 | **최대 100%** |
| **Step 2** | **RAG 기반 압축** | 전체 지식이 아닌, 현재 문장과 관련된 용어/스타일 가이드만 선별하여 프롬프트 주입 | **약 50~70%** |
| **Step 3** | **멀티 타겟 프롬프트** | 단 한 번의 호출로 영/일/중 3개 국어를 동시 결과값(JSON)으로 수신 | **시스템 오버헤드 감소** |

## 3. 주요 기능
- **지식 베이스(KB) 관리**: 엑셀, CSV 등 기존 번역 데이터를 벡터화하여 저장
- **실시간 다국어 번역**: 국문 입력 시 영/일/중 번역 결과 동시 노출
- **토큰 세이빙 대시보드**: 절약된 토큰 양과 비용을 시각적으로 확인
- **피드백 루프**: 수정된 번역본을 실시간으로 지식 베이스에 반영하여 품질 자가 증폭

## 4. 기술 스택 (Tech Stack)
- **Framework**: FastAPI (경량 및 고속 API 처리)
- **Database**: PostgreSQL + pgvector (관계형 데이터와 벡터 데이터 통합 관리)
- **AI Model**: GPT-4o-mini (가성비 모델) + Semantic Cache Layer
- **Infrastructure**: Antigravity Environment

## 5. 실행 로드맵
1. **데이터 전처리**: 기존 번역 데이터 정제 및 벡터 DB 구축
2. **캐싱 로직 구현**: 유사도 기반 검색 엔진 및 캐시 레이어 개발
3. **프롬프트 고도화**: 3개 국어 동시 출력을 위한 JSON 구조 설계
4. **인터페이스 배포**: 사용자 중심의 번역 UI 및 피드백 시스템 구축
